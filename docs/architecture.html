OllamaClaw architecture v0.3.0

A Go agent that runs against an Ollama host. Several ingress channels — REPL, Telegram, cron, GitHub webhooks, a local hotkey server — all funnel into one shared engine backed by SQLite.

01System overview

Five ingress channels feed the same agent.Engine. The engine reads/writes ~/.ollamaclaw/*.db, calls Ollama for chat plus transcription and synthesis, and dispatches to the builtin tool table.

02One turn through the agent loop

Every channel ends up here. The loop is a bounded tool-use cycle (max 12 iterations) with persistence on every step and two background side-effects: compaction once a session crosses the context threshold, and a core-memory refresh every 10 user turns.

  participant U as Ingress

  participant SP as Prompt assembly

+ compaction summary)
  participant O as Ollama /api/chat
  participant T as Tool dispatcher
  participant BG as Background
(compaction + dream)

  U->>E: HandleTextWithOptions(text, images, opts)
  E->>DB: GetOrCreateActiveSession(transport, key)
  E->>DB: InsertMessage(role=user)
  E->>SP: build active prompt
  loop up to 12 iterations
    E->>O: chat(messages, tools, think?)
    O-->>E: assistant reply
(content + tool_calls)
    E->>DB: InsertMessage(role=assistant)
    alt tool_calls present
      E->>T: dispatch each call
      T-->>E: result (or timeout / error)
      E->>DB: InsertMessage(role=tool, per call)
      E-->>U: emit ToolEventStart / Finish
    else final reply
      Note over E: break loop
    end
  end
  E-->>U: assistant text
(chunked, optional voice)
  E->>DB: IncrementSessionTokens
  par if tokens > threshold
    E->>BG: maybeCompact()
summarize + archive
  and every 10 user turns
    E->>BG: refreshCoreMemories()
(non-blocking)
  end

03Tools catalog & dispatch

Tools are constructed once in tools.BuiltinTools() and handed to the engine. Each turn, the combined list is converted to Ollama's tool schema; results route back as role=tool messages and re-enter the loop.

04Background subagents

Two surfaces — generic subagent_start and batched subagent_pr_review — drop tasks into a shared queue. A worker pool (default MaxConcurrent=3) shells out to codex exec, captures stdout/stderr, caps results at 64 KB, and notifies the originating session on completion.

05Integration plug-in points

Telegram is the richest surface — voice, images, slash commands, owner allowlist — but every ingress collapses to a single call into the engine. The hotkey path uses a token-protected loopback HTTP server driven by Hammerspoon; webhooks proactively queue a turn into the owner's Telegram session.
